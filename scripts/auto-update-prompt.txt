ITScape (CVE-2026-46316) is a KVM/arm64 guest-to-host escape in the Linux
kernel: a translation-cache double-put in vgic_its_invalidate_cache() (it
puts the iterated pointer instead of the value returned by xa_erase(), from
mutually-non-exclusive contexts) drops a vgic_irq's cache reference more
than once under concurrency, freeing it while an ITE still maps it — a
use-after-free a guest with EL1 can drive to host-kernel code execution.
The canonical fix is 13031fb6b835 (KVM: arm64: vgic-its: Drop the
translation cache reference only for the erased entry), first released in
v7.1. The bug was introduced by 8201d1028caa in v6.10 (2024-04-25), so the
reachable window is 6.10 through 7.0 and kernels OLDER than 6.10 are NOT
affected. It is arm64-only; the x86 sibling is Januscape (CVE-2026-53359),
tracked separately.
