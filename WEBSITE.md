# Website plan — `https://kimmo.cloud/itscape/`

Publish the ITScape tracker as a single-page static site, generated by
**Hugo** locally and rsync'd to a personal nginx server (`haig`).  The build
infrastructure mirrors the sibling januscape / ipv6_frag_escape / cifswitch
trackers; only the content and a few config strings differ.

## Architecture

- **Source repo:** `github.com/suominen/itscape` (this repo).
- **Generator:** Hugo extended, ≥ 0.146.0.  Run locally; nothing built in
  CI.
- **Source layout:** Hugo project under `site/`.  The tracker is a single
  page at `site/content/_index.md`.
- **Theme:** PaperMod, integrated as a Hugo Module (no submodule).
- **Dev environment:** Nix flake (`flake.nix`) provides hugo + go + git +
  resvg (for `make banner`) + curl/libarchive/zstd (the auto-update agent's
  RPM-unpacking tools).  Auto-activates via `.envrc` if direnv is installed.
- **Build:** `make build` → `hugo --minify --gc --cleanDestinationDir`,
  output in `site/public/`.
- **Publish:** `make dist` → `rsync -avz --delete site/public/` →
  `haig:/itscape/`.
- **Web server:** existing nginx vhost on `kimmo.cloud` serves
  `htdocs/itscape/` directly at the URL path `/itscape/`.

## Naming — nickname slug (CVE assigned)

The bug has a marketable name (**ITScape**) *and* a CVE
(**CVE-2026-46316**).  Following the sibling precedent — which runs both
ways (pintheft and cifswitch kept nicknames; Dirty Frag was renamed to
`/CVE-2026-43284/`) — the site uses the **nickname** as its slug
everywhere, with the CVE carried in the tracker body, Summary, and title:

- repo: `~/src/itscape`, `github.com/suominen/itscape`
- Go module path: `github.com/suominen/itscape/site`
- URL: `https://kimmo.cloud/itscape/`
- systemd units: `itscape-tracker-update.{service,timer}`

No redirect is needed; the slug never moves.

## Cross-reference — the Januscape sibling

ITScape (arm64 vGIC-ITS) and **Januscape (CVE-2026-53359)** (x86
shadow-MMU) are twin KVM guest-to-host escapes disclosed together by Hyunwoo
Kim (`@v4bel`) — the first KVM escape demonstrated on each architecture.
They are architectural complements: ITScape is arm64-only, Januscape
x86-only.  Each tracker cross-links the other in its Summary ("Related"),
the closing note of its intro/commit-range prose, and its References table.
Januscape lives at `~/src/januscape` / <https://kimmo.cloud/januscape/>.

## Tracker content

- One kernel bug, **single verdict axis**: the `13031fb6b835` fix (v7.1) is
  the only thing that flips a row.  There is no userspace reachability gate
  and — because a guest needs its interrupts — no runtime mitigation knob.
- **arm64-only**, introduced in **v6.10**, so the window is narrow (6.10–7.0)
  and many stable EL / Debian releases are genuinely **not affected**
  (contrast the x86 Januscape sibling, whose 2.6.36 origin puts every kernel
  in-window).  The table therefore carries real "not affected — predates the
  trigger (< 6.10)" rows.
- Track the **arm64** build of each distro.  **Proxmox VE is x86-only** and
  is omitted here (covered by Januscape).
- **Distribution status** uses a single **combined table** placed
  immediately under the heading (`Distribution | Release | Kernel | Fixed
  since | Status`), with per-distro `###` headings only where there is an
  audience-relevant note.  Follows the combined-table + sticky "Fixed since"
  layout of the sibling trackers.
- Rows seed from sourced facts: sid/forky/NixOS already carry the fix;
  trixie and Rocky 10 are in-window and vulnerable; pre-6.10 releases are not
  affected.  No unconfirmed "fixed" claims.
- Social/OpenGraph banner — `site/assets/itscape-tracker.svg` rasterised to
  `site/static/itscape-tracker.png` by `make banner` (resvg), wired into the
  `cover:` front-matter.

## Steps

### 1. Hugo project — scaffolded from ipv6_frag_escape

The Hugo skeleton, theme integration, PaperMod overrides, CSS, and i18n were
copied wholesale from the ipv6_frag_escape tracker (via the januscape
sibling) on 2026-07-08.  Only `baseURL`, `title`, the Go module path, the
banner, the systemd unit names, the timer slot, and the rsync destination
were retargeted, and the content + CLAUDE.md + auto-update prompt were
rewritten for this bug.  The auto-update agent's kernel clone is the KVM
tree (`~/src/linux/kvm`), shared with the januscape tracker.

### 2. Publish

- [ ] First `make build` and inspect `site/public/` locally.
- [ ] First `make dist` to push to `haig` — `htdocs/itscape/` created (let
      rsync make it).
- [ ] Verify the site renders at `https://kimmo.cloud/itscape/`, including
      the RSS feed and OG metadata.

### 3. Automated maintenance

- [ ] `git worktree add -b auto-update ~/src/auto-update/itscape main`
- [ ] Install and enable `systemd/itscape-tracker-update.{service,timer}` as
      user units (see CLAUDE.md for the exact `ln -sr` + `systemctl --user`
      recipe).
- [ ] Confirm the timer slot `06,18:05` does not collide with the siblings
      (ipv6 `:05`, cifswitch `:20`, pintheft `:35`, januscape `:50`).

## Decisions

- **Hosting:** own nginx on `haig`, *not* GitHub Pages.
- **URL:** `https://kimmo.cloud/itscape/` (nickname slug; see "Naming").
- **Theme integration:** Hugo Modules; theme PaperMod.
- **Canonical source:** `site/content/_index.md`.
- **Automated maintenance:** a user-level systemd timer
  (`systemd/itscape-tracker-update.timer`, twice daily) runs
  `scripts/auto-update`, which merges `origin/main` into a dedicated
  long-lived `auto-update` branch in a separate worktree and hands off to
  headless Claude with `scripts/auto-update-prompt.txt`.  The agent only
  commits onto `auto-update` — it does not push or open PRs.  Merges of
  `auto-update` into `main` are manual.  The wrapper prints an `ITScape
  tracker auto-update starting …` banner as its first output line so
  aggregated syslog can tell the trackers apart.

## Risks & gotchas

- **Anchor drift.** Hugo/Goldmark's heading slugger differs from GitHub's.
  Pin brittle headings with `### Heading {#stable-id}` if external bookmarks
  need to survive renames.
- **Subpath baseURL.** `baseURL = "https://kimmo.cloud/itscape/"` must
  include the trailing slash and the path component.
- **`.cleanDestinationDir`** wipes `site/public/` before each build.
- **rsync `--delete`** removes server-side files not present in the build
  output.  Don't store unrelated content under `htdocs/itscape/`.
- **Client-side table tweaks.** `layouts/partials/extend_footer.html`
  rewrites tables in the browser (rowspan collapse + status-emoji tagging);
  with JavaScript disabled both are skipped and the table still renders,
  just un-compacted.

## Social banner

`site/assets/itscape-tracker.svg` (1200×630) is rasterised to
`site/static/itscape-tracker.png` by `make banner`, which runs `resvg`.  The
PNG is committed so ordinary `make build` / `make dist` runs need no
rasteriser.  `resvg` needs the Roboto fonts (`fonts-roboto-unhinted` on
Debian).
